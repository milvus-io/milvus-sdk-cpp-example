# Milvus SDK C++ Example - Conan Managed

This example demonstrates how to build a C++ application using the [Milvus C++ SDK](https://github.com/milvus-io/milvus-sdk-cpp) with the SDK itself published as a Conan package.

## Prerequisites

- CMake 3.14 or higher
- C++14 compatible compiler
- Conan 2.x package manager installed (see https://conan.io)
- milvus-sdk-cpp published to a Conan remote

## Configuration

This example uses:
- **milvus-sdk-cpp as a Conan package**: The SDK and all its transitive dependencies (gRPC, Protobuf, Abseil, etc.) are resolved automatically by Conan
- **No FetchContent**: Unlike the other examples, the SDK source is not downloaded during the build

## Dependencies

All dependencies are managed transitively by the `milvus-sdk-cpp` Conan package (see `conanfile.py`):
- milvus-sdk-cpp (which transitively depends on gRPC, Protobuf, Abseil, nlohmann_json, etc.)

## Build Instructions

First, add the Conan remote that hosts the `milvus-sdk-cpp` package:

```bash
# Add the milvus Conan remote (only needed once)
conan remote add default-conan-local2 https://milvus01.jfrog.io/artifactory/api/conan/default-conan-local2

# Verify the remote was added
conan remote list
```

Then build:

`make` is equivalent to `make build`.

```bash
# Default: static link (SHARED=OFF)
make

# Dynamic link — requires a shared variant of milvus-sdk-cpp on the Conan remote
make SHARED=ON

# Clean build artifacts
make clean
```

Or manually:

```bash
# Install Conan dependencies into the build directory
conan install . --output-folder=cmake_build --build=missing -s compiler.cppstd=14 -s:b compiler.cppstd=17 -s build_type=Release

# Configure and build with CMake
mkdir -p cmake_build && cd cmake_build
cmake -DCMAKE_BUILD_TYPE=Release -DCMAKE_TOOLCHAIN_FILE=conan_toolchain.cmake ../
make -j8
```

### SHARED=ON vs SHARED=OFF

- **`SHARED=OFF` (default)**: Uses the default (static) variant of `milvus-sdk-cpp` from the Conan remote. `my_program` is a self-contained binary.
- **`SHARED=ON`**: Passes `-o milvus-sdk-cpp/*:shared=True -o grpc/*:shared=True -o protobuf/*:shared=True -o abseil/*:shared=True` to Conan. **This requires matching shared variants to be published on the Conan remote**; otherwise Conan will fail with "no package matching". Add `--build=milvus-sdk-cpp` (edit `build.sh`) to rebuild from source if needed.

## Run

After successful build:

```bash
make run
```

Or directly:

```bash
./cmake_build/my_program
```

## How It Works

1. **Conan** resolves `milvus-sdk-cpp/2.6.2` and all its transitive dependencies from the remote
2. **CMake** uses the Conan toolchain file to find the installed packages
3. **`find_package(milvus_sdk)`** locates the SDK headers and libraries
4. The example application links against `milvus_sdk` library

## Generated Files

When you run `conan install`, it generates many files. **All of these are placed in the `cmake_build/` directory** to keep your project root clean:

**Generated by Conan (in cmake_build/):**
- `*.cmake` files - CMake package configuration for dependencies
- `*.sh` files - Environment activation scripts
- `CMakePresets.json` - CMake presets file
- Build artifacts and intermediate files

**Generated by Conan (in project root):**
- `CMakeUserPresets.json` - Points to cmake_build/CMakePresets.json (auto-regenerated)

**Source files (keep in version control):**
- `CMakeLists.txt` - CMake configuration
- `conanfile.py` - Conan dependency specification
- `build.sh` - Build script
- `Makefile` - Make targets
- `src/main.cpp` - Source code

To remove all generated files: `make clean`
